PROJECT-WEBVIUM ("Webvium") is a ultra lightweight, material design and full-featured Android Browser.

Webvium was made possible by the Webview and Chromium open source project.

-- HISTORY --
I started this project on Oct 2018 when i was new in Programming World. This project first name is Webview in it first
year i start learning and adding features. Though the app was still in development stage, it doesn't have much feature
but who i am to stop. On it second year 2019, Webview was been renamed to Webvium and its package name was change from
com.android.DROID_MJ.webview to com.droidmj.webvium. In recent years i've learn so much that most of the things i've
learn is on this Project. From compatibility, stability to performance this project is awesome as its name.

One of the challenging things i've did so far is making this app compatible to all material themes android devices.
I admit this entire project was a hell, well since i've updating this app components one by one. Things makes easier,
Adding SQLite as Webvium primary database was a Purgatory since i do need to change a tons of codes. Shared Preferences
acts as the Secondary database while txt as temporary database.

I do encryption and decryption here on this project to protect sensitive content when user export a database or when app
do right data on sdcard. I cannot guarantee that this algorithm is unbreakable but i do my best as a developer.

This project uses java and xml as its core language while txt, php, html, css, and js as second language. I do so much
hard work on this thing that this Project is in High Priority than my School grades for real. One of the things that i
made this app awesome is being able to receive notifications, updates and etc from Server using Github Repository without
using any 3rd party libraries. Well its part of rules that i'll do my best to developed this app without those fancy
libraries. I've able to make this app much lightweight by reducing the usages of png files instead i use xml vectors.

Using xml vectors is one of the reasons why this Project doesn't support Android below 5.
